# Judging Criteria

FOSSHack Jabalpur 2026 evaluates projects based on the **full development process** — not solely the final demo. Judges will review the entire contribution history across the month of March 2026.

---

## Evaluation Framework

Projects are scored across seven criteria. Each criterion carries equal weight unless otherwise specified by the jury.

---

### 1. GitHub Activity Consistency

Judges will review the commit history, issues, and pull requests made throughout the event period (March 1–31, 2026).

**What is evaluated:**
- Regular, meaningful commits over the entire month
- Well-written commit messages
- Use of GitHub features (issues, pull requests, project boards)
- Evidence of iterative development and improvement

**What is penalized:**
- Bulk commits made at the end of the period
- Commit timestamps that suggest artificial activity
- Sparse or inconsistent contribution patterns

---

### 2. Code Quality

**What is evaluated:**
- Readability and structure of the codebase
- Consistent coding style and conventions
- Modular and maintainable code design
- Appropriate use of comments and documentation within code
- Absence of obvious bugs or security vulnerabilities

---

### 3. Documentation

**What is evaluated:**
- Quality and completeness of the README file
- Clear setup and usage instructions
- Contribution guidelines
- Inline code documentation where appropriate
- Architecture or design documentation for complex projects

---

### 4. Problem-Solving Impact

**What is evaluated:**
- Relevance of the project to a real-world problem
- Clarity of the problem being addressed
- Effectiveness of the proposed solution
- Potential for community adoption and long-term use

---

### 5. Completeness

**What is evaluated:**
- Whether the project delivers a working, demonstrable product
- Core features are functional and tested
- Known issues or limitations are documented
- The project is installable or deployable from the repository

---

### 6. Valid FOSS License

**What is evaluated:**
- The repository includes a recognized FOSS license (see [LICENSES/](LICENSES/))
- The license is correctly applied to the codebase
- Third-party code and assets are attributed appropriately
- The project does not include code that conflicts with its stated license

Projects without a valid FOSS license will be disqualified.

---

### 7. Demo Video

**What is evaluated:**
- A video walkthrough of the project (maximum 3 minutes)
- Demonstrates core functionality clearly
- Explains the problem being solved
- Highlights key technical decisions

**Requirements:**
- Maximum duration: 3 minutes
- Must be publicly accessible (YouTube, Vimeo, or similar)
- Link must be included in the project README

---

## Disqualification Criteria

Projects may be disqualified for any of the following reasons:

- Missing or invalid FOSS license
- Core functionality dependent on proprietary APIs
- Evidence of plagiarism or copied code without attribution
- Commit history that does not reflect genuine development
- Crypto / Web3 / Blockchain functionality
- Violation of the Code of Conduct

---

## Jury and Final Decisions

- The jury panel will consist of experienced open-source contributors and representatives from FOSS United
- Jury decisions are final and binding
- Results will be announced at the in-person Demo Day on March 28, 2026

---

## Prize Structure

Prize distribution is determined at the national FOSSHack 2026 level. The national prize pool is ₹5,00,000+. Local host prizes, if any, will be announced separately by the GGITS organizing team.
