# Problem Statements

FOSSHack Jabalpur 2026 participants may build projects under **Open Innovation** across any of the approved categories, or select from the curated problem statements listed below.

All projects must be fully open source, licensed under an approved FOSS license, and must not rely on proprietary APIs for core functionality.

> **Note:** Crypto / Web3 / Blockchain problem statements are not included and such projects are not eligible.

---

## Category 1: AI / ML Applications

### PS-AI-01: Accessible Language Tool for Regional Languages
Build an open-source tool that supports text translation, transliteration, or speech recognition for Indian regional languages (such as Hindi, Marathi, or Bundeli). The tool should be usable offline or with minimal internet dependency.

### PS-AI-02: Open Dataset Annotation Platform
Create a web or desktop application that allows communities to collaboratively label and annotate datasets for machine learning purposes. Must support image, text, or audio annotation with export functionality.

### PS-AI-03: AI-Assisted Code Review Tool
Build a lightweight, local or self-hosted tool that analyzes code repositories and provides automated quality suggestions without sending data to third-party cloud APIs.

---

## Category 2: Developer & Productivity Tools

### PS-DEV-01: Open Source Project Health Dashboard
Create a dashboard that aggregates and visualizes GitHub repository metrics — including commit frequency, open issues, pull request activity, and contributor diversity — to help maintainers assess project health.

### PS-DEV-02: CLI Productivity Toolkit
Build a command-line interface tool that helps developers automate repetitive tasks such as project scaffolding, environment setup, or documentation generation.

### PS-DEV-03: Self-Hosted Code Snippet Manager
Develop a lightweight, self-hostable application for developers to store, search, tag, and share code snippets privately or publicly.

---

## Category 3: Education Technology

### PS-EDU-01: Offline-First Learning Platform
Build an educational platform that functions fully offline and can be deployed in low-bandwidth or no-bandwidth environments. Focus on a specific subject area (mathematics, science, coding, etc.).

### PS-EDU-02: Open Curriculum Management System
Create a tool for educators to organize, share, and collaborate on open educational resources. Must support content versioning and export to standard formats.

### PS-EDU-03: Interactive Coding Practice Environment
Develop a self-hosted, browser-based coding environment where learners can complete structured exercises and receive automated feedback without requiring external judge systems.

---

## Category 4: Social Impact Applications

### PS-SI-01: Community Reporting and Issue Tracking Tool
Build a platform that allows citizens to report local civic issues (e.g., broken infrastructure, public health concerns) and track resolution status. Must be accessible on mobile browsers.

### PS-SI-02: Open Access Public Health Information System
Create a system that aggregates and presents public health data (disease outbreaks, vaccination schedules, health advisories) in a clear, multilingual format for general audiences.

### PS-SI-03: Volunteer and NGO Coordination Platform
Develop an open-source application that helps NGOs and volunteer groups coordinate activities, manage events, and communicate with participants without relying on proprietary platforms.

---

## Category 5: Sustainability & Environment

### PS-ENV-01: Energy Consumption Tracker
Build a personal or household energy monitoring tool that tracks electricity usage, calculates carbon footprint estimates, and provides actionable reduction suggestions.

### PS-ENV-02: Open Environmental Data Visualizer
Create a tool that ingests publicly available environmental datasets (air quality, water quality, deforestation, etc.) and presents them through interactive, accessible visualizations.

### PS-ENV-03: Sustainable Agriculture Decision Support Tool
Develop an application that helps small-scale farmers make data-informed decisions about crop selection, irrigation, or pest management using open datasets and open-source models.

---

## Submission Notes

- You are not required to select a problem statement from this list. Open Innovation submissions are equally valid.
- If you do select a problem statement, reference the PS code (e.g., `PS-AI-01`) in your project README.
- Problem statements describe a challenge area, not a prescribed solution. Creative approaches are encouraged.
