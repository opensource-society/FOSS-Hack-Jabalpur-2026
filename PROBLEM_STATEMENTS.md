# Problem Statements — FOSSHack Jabalpur 2026

Participants may build original projects under Open Innovation or choose from the curated problem statements listed in this document. All projects must be fully open source, licensed under a valid FOSS license, and built within the event period (March 1–31, 2026).

> **Note:** These are suggested problem areas. Participants are encouraged to define the specific scope and approach for their chosen problem.

---

## Category 1: AI / ML Applications

### PS-AI-01 — Accessible Language Tools
Build a tool that uses natural language processing (NLP) to assist users with limited literacy, including text simplification, audio-to-text conversion, or regional language support.

### PS-AI-02 — Smart Agriculture Assistant
Develop an AI-based advisory tool that helps smallholder farmers with crop recommendations, pest detection, or weather-based planting guidance using open datasets.

### PS-AI-03 — Bias Detection in Text
Create a tool that analyzes text content (news articles, job descriptions, etc.) for gender, regional, or socioeconomic bias and provides a structured report.

---

## Category 2: Developer & Productivity Tools

### PS-DEV-01 — Open Source Contribution Tracker
Build a dashboard or CLI tool that helps developers track their open-source contributions across GitHub, GitLab, or other platforms and generate shareable summaries.

### PS-DEV-02 — Dependency License Checker
Develop a tool that scans a project's dependencies and generates a report on license compatibility, flagging any potential conflicts with the project's chosen license.

### PS-DEV-03 — Documentation Generator
Create a tool that parses source code (in one or more programming languages) and automatically generates structured documentation in Markdown or HTML format.

---

## Category 3: Education Technology

### PS-EDU-01 — Offline Learning Platform
Build a lightweight, offline-capable platform for delivering structured learning content, quizzes, and progress tracking — targeting users with limited or intermittent internet access.

### PS-EDU-02 — Collaborative Study Tool
Develop an open-source platform where students can create, share, and collaboratively annotate study notes, mind maps, or flashcards.

### PS-EDU-03 — Skill Assessment System
Create an open-source system for conducting skill-based assessments with automated scoring and detailed feedback, suitable for technical and non-technical subjects.

---

## Category 4: Social Impact Applications

### PS-SOC-01 — Civic Issue Reporting Tool
Build a web or mobile application that allows citizens to report local civic issues (potholes, water supply, electricity failures) and track resolution status using open government data where available.

### PS-SOC-02 — Emergency Resource Finder
Develop an application that helps users locate nearby emergency resources — including hospitals, blood banks, shelters, and helplines — with offline support and low-bandwidth optimization.

### PS-SOC-03 — Community Volunteer Platform
Create a platform that connects volunteers with local NGOs, community organizations, and social causes, including event listings, sign-ups, and impact tracking.

---

## Category 5: Sustainability & Environment

### PS-ENV-01 — Personal Carbon Footprint Calculator
Build an open-source tool that helps individuals calculate and understand their carbon footprint based on lifestyle inputs, with suggestions for reduction using publicly available emissions data.

### PS-ENV-02 — Air Quality Monitoring Dashboard
Develop a dashboard that aggregates and visualizes air quality data from open APIs or community sensors, with alerting and historical trend analysis.

### PS-ENV-03 — Waste Segregation Guide
Create an application that helps users identify and correctly segregate household waste (wet, dry, hazardous, recyclable) using image recognition or a structured lookup system.

---

## Open Innovation

Participants who do not wish to work on a curated problem statement may build any project that:

- Falls within one of the five listed categories.
- Solves a real and identifiable problem.
- Is fully open source with a valid FOSS license.
- Does not rely on Crypto, Web3, or Blockchain technologies.

Participants choosing Open Innovation are still required to clearly document the problem they are solving in their project README.

---

## Submission Requirements

When submitting a project:

1. Clearly state which problem statement (PS code) you are addressing, or indicate "Open Innovation."
2. Include a problem description in your project README.
3. Link your public GitHub repository in your final submission.
4. Ensure all code is committed within the March 1–31 window.
