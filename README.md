## About the Event

FOSSHack Jabalpur 2026 is the Jabalpur local chapter of **FOSSHack 2026**, a national-level Free and Open Source Software (FOSS) hackathon supported by [FOSS United](https://fossunited.org/).

## What is FOSS?

Free and Open Source Software (FOSS) refers to software that is both free (as in freedom) and open source. FOSS is built on the principles of:

- **Open Collaboration** — anyone can view, use, modify, and distribute the software
- **Public Repositories** — source code is hosted publicly, typically on platforms like GitHub or GitLab
- **Licensing** — projects use approved open-source licenses (MIT, Apache 2.0, GPL, etc.) that define usage rights
- **Community Contributions** — improvements are proposed by contributors via pull requests, issues, and discussions

FOSS powers the modern internet. From Linux and Python to Firefox and Kubernetes, the world runs on open source.

## Project Categories

Participants may build projects in any of the following categories:

- AI / ML Applications
- Developer & Productivity Tools
- Education Technology
- Social Impact Applications
- Sustainability & Environment

All categories fall under **Open Innovation**. Participants may also choose from the curated problem statements listed in [PROBLEM_STATEMENTS.md](PROBLEM_STATEMENTS.md).

> **Note:** Crypto / Web3 / Blockchain projects are **not allowed**.

## Eligibility Rules

- The project must be fully open source
- The project must use a valid FOSS license (see [LICENSES/](LICENSES/))
- Core functionality must not depend on proprietary APIs
- All commits must be made within March 1–31, 2026
- Proper attribution must be provided for any third-party code used
- Plagiarism in any form will result in immediate disqualification

---

## How Projects Are Judged

Evaluation is based on the **full development process**, not just the final demo. Judges will review the entire contribution history.

| Criterion | Description |
|---|---|
| GitHub Activity | Consistency of commits, issues, and pull requests throughout March |
| Code Quality | Readability, structure, and maintainability of the codebase |
| Documentation | Quality of README, inline comments, and contribution guides |
| Problem-Solving Impact | Relevance and usefulness of the solution |
| Completeness | Working features and overall project completeness |
| Valid License | Project must include an approved FOSS license |
| Demo Video | A video walkthrough of the project (maximum 3 minutes) |

> **Note:** The jury decision is final and binding.

For full criteria, see [JUDGING_CRITERIA.md](JUDGING_CRITERIA.md).

## How to Contribute

1. Fork this repository
2. Create a new branch: `git checkout -b your-branch-name`
3. Add your project documentation or improvements
4. Commit your changes with a clear message
5. Submit a Pull Request

Please read [CONTRIBUTING.md](CONTRIBUTING.md) before submitting your first pull request.
