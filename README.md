## What is FOSS?

**Free and Open Source Software (FOSS)** refers to software that is both free (as in freedom) and open source. FOSS grants users the freedom to use, study, modify, and distribute the software and its source code.

Key principles of FOSS include:

- **Open Collaboration** — Anyone can contribute to the project regardless of geography or affiliation.
- **Public Repositories** — Source code is publicly hosted and accessible, typically on platforms like GitHub or GitLab.
- **Licensing** — Every FOSS project must carry an approved open-source license (e.g., MIT, Apache 2.0, GPL v3).
- **Community Contributions** — Projects grow through transparent, community-driven development.

## Project Categories

Participants are encouraged to build projects in one of the following categories:

- **AI / ML Applications** — Tools and applications powered by artificial intelligence or machine learning.
- **Developer & Productivity Tools** — Utilities, CLIs, libraries, or platforms that improve developer workflows.
- **Education Technology** — Platforms and tools that improve learning and access to education.
- **Social Impact Applications** — Solutions that address civic, health, or community challenges.
- **Sustainability & Environment** — Projects focused on environmental awareness, energy efficiency, or climate data.

> All categories fall under **Open Innovation**. Participants may also choose from the curated problem statements available in [PROBLEM_STATEMENTS.md](PROBLEM_STATEMENTS.md).

> **Note:** Projects based on Crypto, Web3, or Blockchain technologies are **not allowed**.

## Eligibility Rules

To be considered for evaluation, projects must meet all of the following criteria:

- The project must be **fully open source** and hosted in a public repository.
- The project must use a **valid FOSS license** from the approved list in the [LICENSES/](LICENSES/) folder.
- Core functionality must **not depend on proprietary APIs** or closed-source services.
- All commits contributing to the project must be made **between March 1 and March 31, 2026**.
- Proper **attribution** must be provided for any third-party code, assets, or libraries used.
- **Plagiarism** of any kind will result in immediate disqualification.

---

## How Projects Are Judged

Evaluation is based on the **complete development process**, not just the final demonstration. Judges will review your entire GitHub activity throughout the month.

Projects are evaluated on the following criteria:

| Criterion | Description |
|---|---|
| **GitHub Activity** | Consistency and quality of commits throughout the month |
| **Code Quality** | Readability, structure, and maintainability of the codebase |
| **Documentation** | Quality of README, inline comments, and supporting docs |
| **Problem-Solving Impact** | Real-world relevance and effectiveness of the solution |
| **Completeness** | Functional state of the project at submission |
| **Valid License** | Presence of an approved open-source license |
| **Demo Video** | A clear, concise demonstration video (maximum 3 minutes) |

For full scoring details, refer to [JUDGING_CRITERIA.md](JUDGING_CRITERIA.md).

> **Note:** The jury's decision is final.

## How to Contribute

To submit your project or contribute to this repository:

1. **Fork** this repository to your GitHub account.
2. **Create a new branch** for your changes:
   ```bash
   git checkout -b add/your-project-name
   ```
3. **Add your project documentation** in the appropriate location.
4. **Commit your changes** with a clear, descriptive commit message:
   ```bash
   git commit -m "Add: project documentation for <project-name>"
   ```
5. **Push** your branch and **submit a Pull Request**.

Please read [CONTRIBUTING.md](CONTRIBUTING.md) before submitting a pull request.
